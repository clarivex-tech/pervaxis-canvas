# Pervaxis.Canvas — What It Is and Why It Matters

---

## The Simple Idea

Building a software product today means building two things at once — a website and a mobile app. Most companies build these separately, which means double the cost, double the time, and double the effort every time something needs to change.

**Pervaxis.Canvas solves this.**

Canvas is a pre-built software foundation — a ready-made starting point that every new product is built on top of. Think of it as a professionally designed, fully equipped kitchen. When a new restaurant opens, they do not build a kitchen from scratch. They move into a ready kitchen and focus entirely on their menu.

Canvas is that kitchen. Your product's unique features are the menu.

---

## The Press and Prints Analogy

Imagine a printing press. The press itself never changes — it is a precision machine that produces consistent, high-quality output every time. What changes is what you print.

- **Pervaxis.Canvas** is the press
- **Each customer's product** is a print

When a new customer needs a product, we run the press. Out comes a fully structured, ready-to-build product — web app and mobile app included — in minutes, not months.

---

## What Every Customer Gets

Every product generated by Canvas includes:

**A web application** — accessed through any browser on desktop or laptop computers.

**A mobile application** — available on iOS (iPhone/iPad) and Android devices, built simultaneously from the same codebase as the web app.

Both apps share the same business logic, the same data, and the same security. Building one does not mean doing extra work for the other.

---

## How Multiple Features Are Managed

Large products have many features — customer management, invoicing, reporting, trading, and so on. Canvas organises these as independent building blocks called **modules**.

Each module is built and maintained independently. This means:

- A change to the invoicing module does not affect the customer management module
- A new module can be added to a live product without taking anything offline
- The same module can be shared across multiple customers' products

---

## How a New Customer Is Onboarded

With Pervaxis.Forge — the onboarding engine built on top of Canvas — onboarding a new customer is a single action.

A non-technical team member opens the Forge web interface, enters the customer name, selects which modules they need, and clicks **Onboard**.

The system automatically:
1. Creates the customer's web application
2. Creates the customer's mobile application
3. Sets up all selected modules
4. Organises all code into managed repositories
5. Prepares the product for development to begin immediately

What previously took weeks of setup now takes minutes.

## What Canvas Is Not

Canvas is not the final product. It is the foundation. Canvas handles all the technical infrastructure that every product needs but no customer wants to pay to build repeatedly — security, login, navigation, device compatibility, performance, and structure.

The development team's time and the customer's budget go entirely towards the features that make that customer's product unique.
